bool AcUtils.Transaction.Equals ( Transaction  other)
inline

IEquatable implementation to determine the equality of instances of type Transaction. Compares transactions for equality based on transaction ID, type, time, and principal.

Although transaction ID's are unique within a depot, our Hist container class can hold transactions from multiple depots, thus the potential for duplicate ID's. So instead of a simple transaction ID comparison, we use the approach defined here.

Parameters
otherThe Transaction object being compared to this instance.
Returns
true if Transaction rhs is the same, false otherwise.

The ToString implementation.

Parameters
formatThe format specifier to use, e.g. Console.WriteLine(tran.ToString("LV"));
providerAllow clients to format output for their own types using ICustomFormatter.
Returns
The formatted string.
Exceptions
FormatExceptionthrown if an invalid format string is specified.
Format specifiers:
  • G Transaction ID (default when not using a format specifier).
  • LV Long version (verbose).
  • K Transaction type, e.g. promote, keep, add, etc.
  • T Transaction time.
  • U User (principal) name.
  • S To stream name and number ver_spec, e.g. NEPTUNE_UAT\17 (promote only)
  • F From stream name and number ver_spec (promote only).
